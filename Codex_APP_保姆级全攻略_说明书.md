## 概述

Codex APP 是 OpenAI Codex 四大产品形态中最强的一个。相比 Claude Code，它具有以下优势：

| 对比维度 | Codex APP | Claude Code |
|----------|-----------|-------------|
| 额度 | 更高 | 有限 |
| 功能 | 更全面 | 基础 |
| 上手难度 | 更快 | 需要配置 |
| 免费账户 | 可用 | 不可用 |
| 稳定性 | 无限速/封号/降智 | 存在限速风险 |
| Computer Use | 仅 macOS | 不支持 |


## 第三章 权限控制与沙箱

### 3.1 核心概念

Codex 的权限控制全部围绕**沙箱（Sandbox）**展开。沙箱是整个权限系统运行的地基。

**与 Claude Code 的关键区别**：
- Claude Code：沙箱是额外保护层，需手动开启
- Codex：沙箱是权限系统的基础设施，默认开启

### 3.2 沙箱机制

```
┌─────────────────────────────────┐
│           沙箱（项目文件夹）         │
│  ┌───────────────────────────┐  │
│  │  Codex 可自由读写所有文件    │  │
│  │  无需逐个确认              │  │
│  └───────────────────────────┘  │
│                                   │
│  默认限制：                         │
│  ❌ 不能修改沙箱外的文件             │
│  ❌ 沙箱禁止联网                    │
└─────────────────────────────────┘
```

> 这两点硬性限制使用操作系统底层功能实现（macOS 使用 Seatbelt Sandbox 机制）。这就是 **Harness Engineering** 概念的工程实现——用操作系统级别机制把 AI 能力约束在可控范围内。

### 3.3 权限管理三个档位

| 档位 | 名称 | 说明 | 适用场景 |
|------|------|------|----------|
| 第一档 | **默认** | 提权操作需要人工审核同意 | 新手、敏感项目 |
| 第二档 | **自动审查**（推荐） | 小模型自动审查安全性，低风险放行，高风险人工审查 | 日常开发 |
| 第三档 | **完全访问权限** | 无视沙箱限制，可执行一切操作 | 信任环境、高级用户 |

### 3.4 提权操作（Escalate）

当 Codex 需要修改沙箱外文件或联网时，会向用户申请权限（Escalate）。用户可选择：
- ✅ 批准（单次）
- ✅ 拒绝
- ⚠️ 第三档模式下自动批准所有提权请求

## 第四章 上下文

### 4.1 上下文信息

界面显示当前对话历史内容占用了多少模型上下文空间。

### 4.2 上下文管理

| 操作 | 方式 | 说明 |
|------|------|------|
| 自动压缩 | 超限时自动触发 | Codex 自动压缩对话历史 |
| 手动压缩 | 输入 `/` → 选择"压缩选项" | 排除不重要内容，降低 Token 消耗 |
| 清空（推荐） | 开新对话 | 清空好于压缩，减少干扰 |

> **最佳实践**：完成任务后开新对话。过多历史会话会干扰 AI 注意力。

### 4.3 模型与速度

| 选项 | 说明 |
|------|------|
| **模型选择** | 根据任务复杂度选择思考强度，推荐 GPT-5.5 |
| **标准模式** | 正常推理速度 |
| **快速模式** | 提升 50% 推理速度，但消耗 2 倍套餐用量 |

### 4.4 套餐余量

- **5 小时限额** + **周限额**
- 任意一个到达上限，Codex 不能继续使用
- 各有对应重置时间

### 4.5 语音输入

左下角语音输入功能，支持口述交互。

## 第五章 AI 生图

### 5.1 内置模型

**GPT-Image-2** — 当今最强 AI 生图模型

### 5.2 图片操作

| 操作 | 快捷键 |
|------|--------|
| 粘贴图片到对话 | `Ctrl+V` / `Command+V` |

### 5.3 Steer（引导）功能

当 AI 执行过程中理解错误时，可以及时接管方向盘：

```
正常流程：AI 执行中 → 发送新指令 → 进入指令队列排队
引导流程：AI 执行中 → 点击"引导"按钮 → 立即干预当前执行
```

- 快捷键：`Ctrl+Enter` / `Command+Enter`
- 在执行过程中发送新指令，默认进入**排队**模式
- 点击"引导"按钮可**中途接管**，立即干预

### 5.4 跟进行为设置

**路径**：设置 → 常规设置 → 跟进行为

| 选项 | 说明 |
|------|------|
| 排队（默认推荐） | 新指令进入队列，等当前任务完成 |
| 直接引导 | 新指令立即中断当前任务 |

## 第六章 计划模式与内置浏览器

### 6.1 附加功能（加号菜单）

三个附加功能入口：

```
┌─────────────────────┐
│ 📷 添加照片和文件      │  ← 也可直接 Ctrl+V 粘贴
│ 🔌 插件              │  ← 预装 4 个插件
│ 📋 计划模式           │  ← 先输出计划再执行
└─────────────────────┘
```

**预装插件**：Word、Excel、PPT、浏览器自动化

### 6.2 计划模式

```
开启计划模式 → Codex 输出完整工作计划（不执行）
     ↓
用户确认计划
     ↓
Codex 按计划逐步实施
```

- 倾向使用**问题卡片**形式与用户沟通
- **推荐所有复杂任务都先开启计划模式**
- 问题卡片可让用户做选择（如技术栈选型、架构方案等）

### 6.3 内置浏览器

```
代码编写完成 → 启动内置浏览器 → 自动化测试
                    ↓
         右侧多功能窗口自动打开
                    ↓
         点击"批注"按钮 → 选中元素 → 添加评论 → AI 修改
```

**批注功能**：
1. 点击批注按钮
2. 选中页面上的元素
3. 添加评论/修改要求
4. AI 根据批注自动修改代码
5. 刷新浏览器查看效果

## 第八章 Git 回滚

### 8.1 两个回滚维度

```
完整回滚 = 对话分叉（Fork）+ Git 代码回滚
```

### 8.2 对话分叉（Fork）

```
找到上一次对话结尾位置 → 点击分叉按钮 → 选择"派生到本地"
     ↓
在点击位置复制对话，剔除不需要的部分
```

> ⚠️ 分叉只能回退对话历史，**不能同步回退代码**

### 8.3 代码回滚

```
步骤 1 → VSCode → Source Control → 查看所有 Git 提交
步骤 2 → 右键目标提交 → Copy Commit Hash
步骤 3 → 回到 Codex → "帮我把代码回退到 [commit hash]"
```

### 8.4 完整回滚流程

```
代码回滚（恢复文件状态）+ 对话分叉（恢复对话上下文）= 完整回退
```

## 第九章 Git WorkTree

### 9.1 WorkTree 原理

```
用 Git 创建新分支 → 把新分支代码完整复制到新文件夹

┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   主文件夹     │     │  分支文件夹 A  │     │  分支文件夹 B  │
│  （main）     │←───→│  （feature-a） │←───→│  （feature-b） │
│              │     │              │     │              │
│  并行工作      │     │  并行工作      │     │  并行工作      │
│  互不干扰      │     │  互不干扰      │     │  互不干扰      │
└──────────────┘     └──────────────┘     └──────────────┘
        ↑                   ↑                   ↑
        └─────────── 可随时合并回主干 ─────────────┘
```

### 9.2 操作方式

```
找到项目 → 右键 → 创建永久工作树
```

### 9.3 工作流程

```
步骤 1 → 创建多个 WorkTree 分支（不同文件夹）
步骤 2 → 各分支并行开发
步骤 3 → 开发完毕 → "合并回主干"
步骤 4 → 使用完毕 → 右键移除临时分支
```

## 第十一章 记忆系统

### 11.1 问题

> 每次新对话，AI 不记得之前发生的事。

### 11.2 方法一：AGENTS.md 文件

在项目根目录创建 `AGENTS.md` 文件，每次对话 AI 自动读取此文件作为上下文。

- ✅ **通用格式**：Claude Code、OpenCode 等其他 AI Agent 工具也支持
- ✅ 可让 Codex 帮忙编写

```
提示词：帮我编写 AGENTS.md
```

### 11.3 全局 AGENTS.md

| 项目 | 说明 |
|------|------|
| **路径** | `C:/Users/{用户名}/.codex/AGENTS.md` |
| **设置入口** | 设置 → 个性化 → 自定义指令 |
| **作用范围** | 对所有项目生效 |

### 11.4 AGENTS.md 安全提示词示例

```markdown
禁止使用脚本批量删除文件或目录，只能使用 Remove-Item 一个一个文件进行删除。
如果必须批量删除文件，应该停止操作，让用户手动地批量删除。
```

### 11.5 实验性记忆功能

设置中"启用记忆"开关，可从聊天生成新记忆带入新聊天。

## 第十三章 Skills

### 13.1 Skills 定义

> Skills 是给 AI Agent 的**专业技能包**——带目录的说明书。
> 可封装工作流、专业能力或规范。

### 13.2 三种安装方式

#### 方式一：官方 Skills

```
入口：插件与技能 → 官方技能
安装后用 "/" 唤起使用
```

**示例**：Remotion（编程方式创建动画视频）

#### 方式二：第三方 Skills

```
步骤 1 → 从 GitHub 下载 zip 文件
步骤 2 → 解压到 项目根目录/.codex/skills/ 文件夹
步骤 3 → 用 "/" 唤起使用
```

**示例**：歸藏开发的电子杂志风网页 PPT skill

#### 方式三：自创建 Skills

```
输入 "/" → Skill Creator → 按提示编写技能
换行用 Shift+Enter（不发送）
产出：SKILL.md 文件 + 辅助脚本
```

**示例**：视频字幕转 Markdown 技能（含 ffmpeg 截图占位符功能）

### 13.3 Skills 文件结构

```
项目根目录/
```

```
步骤 6 → 浏览器授权
步骤 7 → ⚠️ 重启 Codex（必须）
```
